========================================================
  用 Pydantic BaseModel 优化智能体中间变量 —— 修改计划
========================================================

一、现状分析
-----------
当前项目使用 TypedDict 定义 LangGraph 状态，存在以下问题：

1. main_agent.py:  VisionAgentState (TypedDict)
   - analysis_results: Optional[dict]  →  完全无类型约束，存的是整个 AgentState 字典

2. analysis_agent.py:  AgentState (TypedDict)
   - final_response: str  →  存的是原始 JSON 字符串，需要手动 json.loads + 清理 Markdown

3. vision_agent.py:  VisionResponse (BaseModel)  ✅ 已经是 Pydantic，但定义在局部文件中

4. api/main.py:  full_state.get("analysis_results", {}).get("final_response")
   →  多层 .get() 取值，无类型保障，容易拼错键名


二、目标
--------
1. 将所有 TypedDict 状态替换为 Pydantic BaseModel（LangGraph 原生支持）
2. 将 final_response 从原始 JSON 字符串改为结构化 Pydantic 对象
3. 将 summarize_llm 改用 with_structured_output() 直接输出结构体，消除手动 JSON 解析
4. 将所有共享模型集中到 backend/models/schemas.py


三、涉及文件
-----------
- backend/models/schemas.py          （新建/扩充，集中放所有 Pydantic 模型）
- backend/app/agents/vision_agent.py  （移动 VisionResponse，其余不变）
- backend/app/agents/main_agent.py    （VisionAgentState TypedDict → BaseModel）
- backend/app/agents/analysis_agent.py（AgentState TypedDict → BaseModel，
                                        summarize_node 改结构化输出）
- backend/app/api/main.py             （改为直接属性访问）


四、详细修改步骤
===============

【步骤 1】backend/models/schemas.py  —— 定义所有共享模型
--------------------------------------------------------

    from __future__ import annotations
    from typing import Annotated, List, Optional
    import operator
    from pydantic import BaseModel, Field


    # ── 1. 视觉节点输出（从 vision_agent.py 移过来）
    class VisionResponse(BaseModel):
        is_valid: bool = Field(description="图片是否清晰且包含食物")
        reason:   str  = Field(description="不合法时说明原因")
        report:   str  = Field(description="识别到的食材详细报告，不合法则为空")


    # ── 2. 营养宝塔各层结构
    class L5Detail(BaseModel):
        oil:  float = 0.0
        salt: float = 0.0

    class PagodaLayer(BaseModel):
        total_value: float     = 0.0
        ingredients: List[str] = []
        details:     dict      = {}

    class PagodaNutritionVector(BaseModel):
        L1: PagodaLayer = Field(default_factory=PagodaLayer)
        L2: PagodaLayer = Field(default_factory=PagodaLayer)
        L3: PagodaLayer = Field(default_factory=PagodaLayer)
        L4: PagodaLayer = Field(default_factory=PagodaLayer)
        L5: L5Detail    = Field(default_factory=L5Detail)


    # ── 3. 分析节点最终输出（替代原来的 JSON 字符串）
    class NutritionReport(BaseModel):
        dish_name:               str                   = ""
        main_ingredients:        List[str]              = []
        seasonings:              List[str]              = []
        pagoda_nutrition_vector: PagodaNutritionVector = Field(default_factory=PagodaNutritionVector)
        feature_tags:            List[str]              = []
        description:             str                   = ""


    # ── 4. 主图谱 LangGraph 状态（替代 VisionAgentState TypedDict）
    class VisionAgentState(BaseModel):
        username:         str                       = ""
        image_path:       str                       = ""
        error_reason:     Optional[str]             = None
        vision_report:    Optional[VisionResponse]  = None
        analysis_results: Optional[NutritionReport] = None


    # ── 5. 分析子图 LangGraph 状态（替代 AgentState TypedDict）
    class AnalysisState(BaseModel):
        username:       str                           = ""
        vision_report:  str                           = ""
        extracted_info: str                           = ""
        messages:       Annotated[list, operator.add] = []
        final_response: Optional[NutritionReport]    = None   # ← 不再是字符串！
        retry_count:    int                           = 0
        errors:         List[str]                     = []
        save_status:    bool                          = False

【步骤 4】analysis_agent.py  —— 最重要的改动
--------------------------------------------

4-a. 状态替换
    修改前：
        class AgentState(TypedDict): ...

    修改后：
        from models.schemas import AnalysisState, NutritionReport
        # 删除本地 AgentState，将所有引用替换为 AnalysisState

4-b. summarize_node 改用 with_structured_output，消除 JSON 解析
    修改前：
        self.summarize_llm = get_llm()
        ...
        response = self.summarize_llm.invoke(messages)
        clean_json = response.content.replace("```json", "").replace("```", "").strip()
        return {"final_response": clean_json}

    修改后：
        self.summarize_llm = get_llm().with_structured_output(NutritionReport)
        ...
        report: NutritionReport = self.summarize_llm.invoke(messages)
        return {"final_response": report}   # 直接存结构体

4-c. 状态访问改为属性访问
    修改前：  state.get("vision_report", "")  /  state["extracted_info"]
    修改后：  state.vision_report              /  state.extracted_info

4-d. analyze() 入口函数
    修改前：
        initial_state = {
            "username": username,
            "vision_report": vision_report,
            "messages": [],
            "retry_count": 0,
            "errors": []
        }

    修改后：
        initial_state = AnalysisState(
            username=username,
            vision_report=vision_report
        )
        return self.workflow.invoke(initial_state)


【步骤 5】api/main.py  —— 改为属性访问
--------------------------------------
    修改前：

        result = full_state.get("analysis_results", {}).get("final_response")

五、改动收益汇总
================
| 改动点                          | 改动前               | 改动后                         |
|---------------------------------|----------------------|--------------------------------|
| LangGraph 状态定义              | TypedDict            | Pydantic BaseModel（带默认值） |
| final_response 类型             | str（原始 JSON）     | NutritionReport（结构化对象）  |
| summarize_node JSON 解析        | 手动 replace+loads   | with_structured_output 自动    |
| 状态字段访问                    | state["key"]         | state.key（属性访问）          |
| 模型定义位置                    | 分散在各 agent 文件  | 统一在 models/schemas.py       |
| API 返回值                      | 字符串二次解析       | report.model_dump() 直接序列化 |


六、注意事项
============
1. LangGraph 版本：langgraph >= 0.1.x 原生支持 Pydantic BaseModel 作为 StateGraph 状态。
   Annotated[list, operator.add] 在 BaseModel 中同样有效。
   节点函数返回 dict，LangGraph 会自动 merge 到当前状态。

2. with_structured_output 要求：
   模型需支持 function calling 或 JSON mode。
   SUMMARIZE_NODE_PROMPT 中的 JSON 示例可保留，但不再需要要求模型输出代码块。

3. 建议在 NutritionReport 中添加：
   model_config = ConfigDict(extra="ignore")  # 忽略 LLM 输出的多余字段

4. 测试时优先验证 analysis_agent.py 的 summarize_node，
   确认 with_structured_output(NutritionReport) 能正确解析 LLM 输出。


七、建议修改顺序
================
Step 1 → schemas.py         （建好基础模型）
Step 2 → vision_agent.py    （最简单，只移动导入）
Step 3 → analysis_agent.py  （核心改动）
Step 4 → main_agent.py      （依赖 analysis_agent 的类型）
Step 5 → api/main.py        （最外层）
